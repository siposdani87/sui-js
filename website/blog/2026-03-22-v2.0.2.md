**v2.0.2** brings dark mode fixes, new component features, and internal modernization.

<!-- truncate -->

### New features

- **fabButton helper** — separated from `iconButton`; `createFabButton` and `fabButton` for FAB-style buttons, `iconButton` is now icon-only by default
- **Label auto-capitalization** — field labels automatically capitalize the first character
- **Button variants showcase** — example page now includes flat, raised, icon, FAB, and icon+text button demos
- **State.setParam type widened** — accepts `string[]` and `number[]` in addition to primitives

### Dark mode & style fixes

- Resolve hardcoded colors breaking dark mode in select options and contentHandler
- Improve dropdown menu hover effect for both light and dark themes
- Replace hardcoded colors with CSS custom properties across button, spinner, slider, menu, and footer styles
- Restore static footer text colors to match v1 design
- Swap card background colors for better default/hover contrast
- Reduce switch component size
- Align modal title SVG and text vertically with gap
- Reorder z-index values so progress bar correctly overlays nav and header

### Internal improvements

- Migrate HTML parsing to DOMParser in navigation, dialog, and template
- Add null safety checks to Knot DOM methods (appendChild, removeChildren, hasChildren, beforeChild)
- Use absolute asset paths in example page
