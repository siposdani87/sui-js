# Quick Start Pattern
```typescript
import { Application, Route } from '@siposdani87/sui-js';

const config = {
    app_id: 'my-app',
    locale: 'en-GB',
    backend: 'https://api.example.com',
    production: false,
};

const routes: Route[] = [
    { id: 'home', url: '/', controller: 'homeCtrl', template: 'home.html' },
];

const app = new Application(config);
app.run(routes, ['homeCtrl']);
```

# Common Patterns
- Constructor: constructor(opt_options) -> _setOptions() -> _init()
- DOM: new Knot(selector) or new Query('.selector').getKnot()
- HTTP: http.get/post/put/patch/delete(url, data?, headers?)
- Events: eventBus.on(event, handler) / eventBus.trigger(event, data)
- Forms: form.load(data) / form.getModel()
- State: state.go(routeId, params)
- Storage: depot.set/get/remove(key, value)
- Cookies: cookie.set/get/remove(name, value)

# When Helping Users
1. Always import from '@siposdani87/sui-js'
2. Application must be initialized first via new Application(config).run(routes, services)
3. Use Knot for DOM manipulation, not raw DOM APIs
4. Use Objekt for data objects, not plain objects
5. Use Material Design Lite classes for styling
6. All async operations return Promize (custom Promise wrapper), not native Promise
7. Optional parameters use opt_ prefix (e.g., opt_options)
8. Private members use _ prefix (e.g., _init())
