# Quick Start Pattern
```typescript
import { Application, Route } from '@siposdani87/sui-js';

const config = {
  app_id: 'APP_ID',
  locale: 'en-GB',
  backend: 'https://api.example.com',
  production: false
};

const routes: Route[] = [];
const services: string[] = [];

const app = new Application(config);
app.run(routes, services);
```

# Core Components (Most Important)
1. Application - Main application class for initialization
2. Knot - DOM manipulation wrapper (jQuery-like)
3. Http - HTTP client with promise support
4. Router - Client-side routing
5. State - Reactive state management
6. Form - Form handling and validation
7. TextField, SelectField, etc. - Form inputs

# Architecture Layers
1. Core (Knot, State, Router, Collection, Module)
2. Modules (Http, EventBus, Dialog, Flash, Depot, Cookie)
3. Components (60+ UI components like Table, Calendar, Header)
4. Fields (Form inputs: Text, Select, Date, Number, etc.)

# Common Patterns
- Initialization: new Application(config)
- DOM: new Knot(selector) or Query.select(selector)
- HTTP: http.get/post/put/delete(url, data)
- Events: eventBus.on/trigger(event, data)
- Forms: form.load(data) / form.getModel()
- State: state.set/get/subscribe(key, value)
